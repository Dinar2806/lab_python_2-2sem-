### Дескрипторы в Python

**Дескриптор** — это объект класса, который определяет, как будут получаться, устанавливаться или удаляться атрибуты другого класса. Дескрипторы позволяют перехватывать доступ к атрибутам, добавляя логику валидации, логирования или вычислений.

В Python различают два типа дескрипторов:
1.  **Data Descriptors**: Реализуют методы `__get__` и `__set__`. Имеют приоритет над словарем экземпляра (`__dict__`). Используются для строгой валидации данных (например, проверка диапазона чисел).
2.  **Non-Data Descriptors**: Реализуют только метод `__get__`. Не имеют приоритета над `__dict__`. Если в экземпляре создать атрибут с таким же именем, он "перекроет" дескриптор. Часто используются для кэширования или предоставления информации только для чтения.

### Инкапсуляция и защита инвариантов

**Инкапсуляция** — механизм сокрытия внутренней реализации объекта от внешнего мира. В контексте данной работы она достигается за счет:
*   Использования приватных атрибутов (с префиксом `_`) для хранения реальных данных.
*   Предоставления публичного доступа только через дескрипторы и свойства (`@property`).

**Инвариант класса** — условие, которое должно быть истинным для любого корректного экземпляра класса в любой момент времени. Например:
*   `priority` всегда находится в диапазоне [1, 10].
*   `status` всегда является допустимым значением перечисления `TaskStatus`.
*   `id` никогда не равен `None`.

Нарушение инварианта приводит к генерации специализированных исключений, что предотвращает перевод объекта в некорректное состояние.

### Property vs Descriptors

*   `@property` используется для создания вычисляемых атрибутов или геттеров/сеттеров для конкретных полей внутри одного класса.
*   **Дескрипторы** более универсальны: один дескриптор может быть использован для валидации множества разных атрибутов в разных классах, обеспечивая переиспользование кода валидации.

### Duck typing и контрактное программирование

**Duck typing** — концепция в программировании, основанная на принципе: "Если это выглядит как утка и крякает как утка, то это утка". В контексте данной работы это означает, что объект считается источником задач, если у него есть метод `get_tasks()`, независимо от его типа и наследования.

**Контрактное программирование** — подход к проектированию, при котором компоненты взаимодействуют на основе четко определенных соглашений (контрактов). В Python контракты описываются с помощью `typing.Protocol`.

### Protocol и runtime_checkable

`Protocol` (из модуля `typing`) позволяет определить интерфейс, которому должен соответствовать объект. Декоратор `@runtime_checkable` дает возможность использовать `isinstance()` для проверки соответствия протоколу во время выполнения.

```python
from typing import Protocol, runtime_checkable
from src.task import Task

@runtime_checkable
class TaskSourceProtocol(Protocol):
    def get_tasks(self) -> list[Task]:
        """Получить список задач из источника"""
        ...
```
